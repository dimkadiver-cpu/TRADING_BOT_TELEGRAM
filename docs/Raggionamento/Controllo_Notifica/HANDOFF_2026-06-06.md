## Cosa è stato definito (sessione completa)

### Template — copertura completa

| Gruppo | Event type | File esempi |
|--------|-----------|-------------|
| Signal | SIGNAL_ACCEPTED, SIGNAL_REJECTED, REVIEW_REQUIRED | `Clean_log_signals.md` |
| Entry | ENTRY_OPENED, ENTRY_UPDATED, ENTRY_CANCELLED | `Clean_log_entry_entry_update.md` |
| Close finale | SL_FILLED, TP_FILLED_FINAL, POSITION_CLOSED, BE_EXIT | `Clean_log_position_closed_partial_closed.md` |
| Update | UPDATE_DONE, UPDATE_PARTIAL, UPDATE_REJECTED | `Clean_log_updates.md` |
| Chiusure parziali | TP_FILLED, PARTIAL_CLOSE_EXECUTED | `Clean_log_tp_partial.md` |
| Semplici | PENDING_ENTRY_EXPIRED, REENTRY_ACCEPTED, CANCEL_FAILED, RECONCILIATION_WARNING, RECONCILIATION_FIXED | `Clean_log_simple_notifications.md` |
| Multi-chain | MULTI_CHAIN_SUMMARY, MULTI_CHAIN_UPDATE, MULTI_CHAIN_CLOSED | `Clean_log_multi_chain.md` |

### Unificazioni chiave

- `_CLOSED_BLOCKS` — 4 tipi chiusura finale (SL/TP_FINAL/POSITION/BE_EXIT), transform diverso
- `_UPDATE_BLOCKS` — 3 tipi update (DONE/PARTIAL/REJECTED), transform diverso
- `_PARTIAL_RESULT_BLOCKS` — **nuovo questa sessione** — unifica TP_FILLED e PARTIAL_CLOSE_EXECUTED
  - `_show_value=True` per TP (mostra `Value:`), `False` per PARTIAL_CLOSE
  - stesso pattern di `_CLOSED_BLOCKS`

### Modifiche spec principali (sessione 2026-06-07)

- `_TP_PARTIAL_BLOCKS` + `_PARTIAL_CLOSE_BLOCKS` → `_PARTIAL_RESULT_BLOCKS` unificato
- Header MULTI_CHAIN: `-` → `—` (`UPDATE APPLICATO —`)
- Tabella mapping tre event type MULTI_CHAIN + `summary_kind`
- Typo fix: `"Requires manual review required"` → `"Requires manual review"`
- Nota architetturale "Partial close" semplificata (rimanda a sezione unificata)
- TEMPLATE_REGISTRY aggiornato per entrambi i cambiamenti

### Modifiche file template (sessione 2026-06-07)

- `Clean_log_signals.md` — aggiunto REVIEW_REQUIRED
- `Clean_log_entry_entry_update.md` — aggiunto ENTRY_CANCELLED (2 casi)
- `Clean_log_simple_notifications.md` — creato ex novo (5 tipi), corretto `signal_link` su tutti i chain-specific
- `Clean_log_tp_partial.md` — riscritto come "Chiusure parziali" unificato (6 casi: 3 TP + 3 PARTIAL CLOSED)
- `Clean_log_multi_chain.md` — creato ex novo (5 casi: DONE/PARTIAL, CLOSE_FULL, ERROR/SKIPPED, MOVE_SL_TO_LEVEL, scope globale)

### Modifiche codice (entry_gate.py)

- `MOVE_STOP` migrato da `display_lines` a `changed` dict `{"field": "SL", "old", "new", "note"}`
- `failed_actions: list[dict]` costruito da NOOP events con `removeprefix("NOOP_")` e reason per action
- `rejected_actions` derivato da `failed_actions` (nomi puliti, senza `NOOP_` prefix)
- `display_lines` rimosso dal payload di `_write_update_clean_log`
